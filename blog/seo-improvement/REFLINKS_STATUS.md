## 別PC再開時の手順

1. 最新のリポジトリをpull
2. `.env`（HATENA_ID, HATENA_API_KEY, HATENA_BLOG_DOMAIN）が`youtube-slides/食事指導シリーズ/_shared/.env`にあることを確認
3. 現状の残数を再監査:
   ```bash
   python blog/seo-improvement/audit_reflinks_v2.py 2>&1 | tee blog/seo-improvement/audit_after_v2.log
   ```
4. 継続手段（残561件向け）:
   - **日本語学会GL辞書**: 残の多くは日本語の学会GL/厚労省/WHO日本語訳など。学会名→公式URLの辞書を作って自動リンク
   - **手動上位対応**: クマ被害(27)、生成AI規制(16)、易怒性(15)など医学外/Webリソース中心は手動が確実
   - **PubMed v2 緩和強化**: THRESHOLDを0.65に下げる、または日本語タイトルもJ-STAGE検索する

## 真因メモ

- **誤マッチ対策**: v2 初期版で score=1.0 が「先頭18文字一致」で誤発火（"Gene therapy approaches"→"Mitochondrial disease in adults"）。先頭40文字一致に厳格化、25文字一致時はtoken coverage>=0.4併用に修正
- **build_query 拡張**: 著者なしの「Title始まり」形式（ミトコ記事に多い）に対応するため、extract_titleを2系統化、B戦略（著者なしtitle+年）/B2戦略（短縮title）を追加
- 残561件は学会GL・厚労省・JBpress等のWebリソースが中心で、PubMed検索では原理的に解決不可
