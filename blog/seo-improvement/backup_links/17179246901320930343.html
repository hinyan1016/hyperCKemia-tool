1. 結論：どちらか一方ではなく「使い分け」が正解

最初に結論を書いておきます。

📋 基本方針
「見出し＝体言止めOK」／「中身・結論＝文章で書く」が基本方針と考えるとバランスが良いです。

体言止め（名詞で終わる形）

  • ✅ タイトルや大見出しでは「積極的に使ってよい」
  • ⚠️ 本文・結論では乱用すると、何を言いたいか分かりにくくなる

文で書く（述語まで書く形）

  • ✅ 本文の要点・結論・メッセージは「基本こちらが推奨」
  • 特に「まとめ」「Take home message」は、そのまま読み上げられる文にした方がよい

2. 体言止め・文、それぞれの長所と短所

体言止めの長所・短所

長所

  • 短くてパッと目に入る
  • タイトルバーに収まりやすく、レイアウトがきれい
  • 「症例」「画像所見」「治療方針」など、章立てのラベルとしては非常に使いやすい

短所

  • 「結局何を言いたいのか」が曖昧になりやすい
  • 例）「治療方針」だけだと
    • 何を行ったのか
    • 何が推奨されるのか
    • 著者はどう考えているのか
    が伝わらない
  • 聴衆が口頭説明を聞き逃すと意味が残らない
  • スライドだけ後から見返したときに、論理の流れが追いにくい

文で書くことの長所・短所

長所

  • 「誰が」「何に対して」「どうした／どうだった／どう考える」が明確になる
  • 疾患の病態や因果関係を示したい内科系の学会では、論理性が伝わりやすく高評価
  • スライドだけ見ても、メッセージが独立して理解できる

短所

  • 行数が増えやすく、詰め込みすぎると読みにくい
  • デザイン的にはやや"文字が多いスライド"に見えることもある

3. どの場所で「体言止め」「文」を使うか

ざっくり整理すると、次のようなイメージです。

スライドの要素 体言止め 文（述語まで） コメント
スライドタイトル ◎ 推奨 △ 「症例紹介」「画像所見」でOK
セクション見出し（症例・検査など） ◎ 推奨 △ ラベルとして体言止めでよい
箇条書きの本文 △ ほどほど ◎ 推奨 「文として読める形」を意識
図表のキャプション △ 場合による ○ 「〜を示す」など文だと親切
まとめ・Take home message ✕ 非推奨 ◎ 強く推奨 そのまま読み上げられる文に

4. よくある例と書き換え

患者背景スライド

⚠️ ありがちな書き方（体言止め＋箇条書き）
  • 75歳男性
  • 高血圧、糖尿病
  • てんかんの既往
  • 独居

一見問題なさそうですが、これを少し「文」に寄せると、読み手の負担が減ります。

📋 おすすめの書き方
75歳男性。高血圧・糖尿病・てんかんの既往あり。独居。

→ 箇条書きでも構いませんが、1行ごとが文として読める形を意識すると、スッと頭に入ります。

結論スライド

⚠️ もったいない例（体言止めだけ）
  • 今回の症例の特徴
  • 早期治療の重要性
  • 地域連携の必要性

これだと、「結局どういう話だったのか」が聴衆のなかで整理されません。
📋 より良い書き方
  • 本症例では、○○が背景にあり、△△が誘因となったと考えられた。
  • 早期に□□を行うことで、××を回避できた可能性がある。
  • 同様の症例では、地域の○○科との連携が予後改善に重要と考えられる。

→ 多少長くなっても、まとめは"日本語としての文"で書いた方が、発表後に印象に残ります。

5. 忙しい臨床医向け「運用ルール」

日々の診療の合間にスライドを作ることを考えると、あまり細かいルールを意識するのは現実的ではありません。以下のような簡単ルールを頭に置いておくと、迷いが減ります。

ルール① タイトル・大見出しは体言止めでOK

「症例」「検査所見」「診断」「治療経過」「考察」「まとめ」など、章タイトルは体言止めで問題ありません。むしろ短くシンプルな方が良いです。

ルール② 本文の箇条書きは「文として読める形」を基本

1行が

  • 「○○であった」
  • 「○○と考えられた」
  • 「○○を施行した」

のように、述語まで入っているかを確認する。

⚠️ 注意
体言止めが3行も4行も続くと、「単語メモ」を映しているだけの印象になります。

ルール③ まとめ・Take homeは"そのまま読み上げ原稿にできる"レベル

スライドの「まとめ」を

  • 口に出して読んでみて
  • そのまま発表原稿として違和感がないか

をチェックすると、過不足の確認になります。

6. よくある疑問へのQ&A

Q1. 査読や座長の立場から見て、体言止めが多いと減点になりますか？

「体言止めだから減点」ということはまずありません。

ただし、結論やメッセージが名詞の羅列だけだと

  • 「何を主張したいのか分かりにくい」
  • 「ロジックが弱く見える」

という印象は持たれやすいです。

📋 ポイント
評価されるのは「論理の明確さ」であり、そのために「文で書く」方が有利、というイメージです。
Q2. 英語スライドの場合も同じですか？

基本は同じです。

見出し：

  • "Case presentation"
  • "Imaging findings"
  • "Conclusion"

など名詞句でOK

本文・結論：

  • "We considered that 〜."
  • "This case suggests that 〜."

のように文で書く方が国際的にも一般的です。

Q3. スライド枚数制限が厳しい場合は？

枚数が限られていると、どうしても文字を削りたくなりますが、
削るのは「余計な修飾語」であって、「述語」ではありません。

例：

  • 「非常に強く示唆された」→ 「示唆された」
  • 「きわめて重要であると考えられる」→ 「重要である」

のように、文を短くする工夫で対応するのがおすすめです。

7. まとめ：最低限ここだけ押さえておけば安心

最後に、この記事のポイントを整理します。

📋 重要ポイント

1. 体言止めは「見出し・ラベル」用
スライドタイトルや章立てにはむしろ向いている。

2. 本文・結論は「文」で書く方が安全
とくに「まとめ」「Take home message」は、そのまま読み上げられる文にする。

3. 箇条書きも"日本語として読めるか"をチェック
主語＋述語のセットが見えるように。

4. 迷ったら
  • 「ここだけは絶対に伝えたい」スライド → 文で書く
  • そうでないラベリング部分 → 体言止めOK

このあたりを押さえておけば、「読みやすく、論理が伝わり、査読者・座長にも安心されるスライド」に自然と近づきます。
